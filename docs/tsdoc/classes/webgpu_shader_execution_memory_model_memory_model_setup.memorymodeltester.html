Class MemoryModelTester

Implements setup code necessary to run a memory model test. A test consists of two parts: 1.) A test shader that runs a specified memory model litmus test and attempts to reveal a weak (disallowed) behavior. At a high level, a test shader consists of a set of testing workgroups where every invocation executes the litmus test on a set of test locations, and a set of stressing workgroups where every invocation accesses a specified memory location in a random pattern. 2.) A result shader that takes the output of the test shader, which consists of the memory locations accessed during the test and the results of any reads made during the test, and aggregate the results based on the possible behaviors of the test.
